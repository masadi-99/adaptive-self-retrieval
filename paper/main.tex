\begin{abstract}
Time series foundation models (TSFMs) produce competitive zero-shot forecasts but lack mechanisms for leveraging dataset-specific structure at inference time. We introduce Consensus-Gated Retrieval Correction (\method), a training-free post-hoc method that refines any black-box forecasting model using only its prediction function. Given a query window, \method retrieves $k$ similar historical windows from the series' own training data, normalizes each to match the current context distribution, re-forecasts from each, and computes per-timestep correction residuals. Corrections are applied only at timesteps where a qualified majority of neighbors agree on the correction direction---a \emph{consensus gate} that suppresses noisy adjustments and preserves the base model's predictions where retrieval evidence is ambiguous. \method requires zero learned parameters, zero model internals, and zero additional training. Across 7 standard benchmarks and 3 foundation models (Chronos-Bolt Small, Chronos-Bolt Base, TimesFM), \method improves MSE by 5--9\% on 5 of 7 datasets, with 14 of 28 dataset-horizon pairs reaching statistical significance ($p < 0.05$; 9 survive Benjamini-Hochberg FDR correction). Ablations confirm that consensus gating---not retrieval volume---drives the gains: ungated correction frequently harms performance, while the consensus gate produces a 6.3 percentage-point accuracy gap between high-agreement and low-agreement windows.
\end{abstract}

%===========================================================================
\section{Introduction}
%===========================================================================

Time series foundation models (TSFMs) such as Chronos~\citep{ansari2024chronos}, TimesFM~\citep{das2024timesfm}, and Moirai~\citep{woo2024moirai} achieve strong zero-shot forecasting performance across domains by pretraining on large time series corpora. However, at inference time, these models produce fixed predictions that cannot adapt to the specific distributional structure of the target series. A practitioner deploying a TSFM on a new dataset receives the model's generic forecast with no mechanism for leveraging the series' own history to improve accuracy.

Recent work has explored retrieval-augmented approaches to address this limitation. TS-RAG~\citep{ning2025tsrag} uses a learned Adaptive Retrieval Mixer to fuse retrieved patterns with the model's internal representations. RAFT~\citep{han2025raft} trains retrieval-augmented projections. RAF~\citep{tire2024raf} concatenates retrieved contexts as input prefixes. PIR~\citep{pir2025} uses learned uncertainty estimators for post-hoc revision. These methods require either additional training, access to model internals, or both---limiting their applicability to the increasingly common setting where TSFMs are deployed as black-box prediction APIs.

We ask: \emph{can a fully training-free, black-box method---one that interacts with the model only through its predict() function---systematically improve forecast accuracy?}

We introduce \textbf{Consensus-Gated Retrieval Correction} (\method), which answers this affirmatively. The key insight is that retrieval-based correction is unreliable when applied uniformly: on some timesteps, retrieved neighbors provide useful correction signals, while on others they introduce noise. \method addresses this through a \emph{consensus gate}: at each timestep $t$, corrections are applied only when a supermajority of $k$ retrieved neighbors agree on the direction of error. When neighbors disagree, the base forecast is preserved unchanged.

This design produces three desirable properties: (1)~\method improves accuracy on datasets with recurring patterns where neighbors provide consistent correction signals; (2)~it gracefully degrades to the base forecast on datasets where retrieval evidence is ambiguous; and (3)~the consensus rate itself serves as a diagnostic for whether \method will help on a given series.

Our contributions are:
\begin{enumerate}
    \item \textbf{Consensus gating as a correction principle.} We formalize per-timestep directional agreement among retrieved neighbors as a gate for applying corrections, showing it is both necessary (ungated correction frequently harms) and sufficient (the gate alone, applied to simple median corrections, matches more complex alternatives).
    \item \textbf{A fully black-box, training-free retrieval-correction framework.} \method requires only predict() access to the base model, with zero learned parameters, zero gradients, and zero architecture knowledge.
    \item \textbf{Systematic empirical evaluation across multiple FM families.} We evaluate on 7 benchmarks, 4 horizons, and 3 foundation models (28 model-dataset pairs), with paired $t$-tests, multiple-comparison correction, ablations, cross-model validation, and honest failure analysis.
\end{enumerate}

%===========================================================================
\section{Method}
\label{sec:method}
%===========================================================================

\subsection{Problem Setting}

Given a frozen foundation model $\mathcal{M}$ accessible only through a prediction function $\hat{y} = \mathcal{M}(\mathbf{x})$ that maps a context window $\mathbf{x} \in \mathbb{R}^L$ to a forecast $\hat{y} \in \mathbb{R}^H$, and a training set $\mathcal{D}_\text{train}$, our goal is to produce a corrected forecast $\tilde{y}$ that improves upon $\hat{y}$ without modifying $\mathcal{M}$.

\subsection{Knowledge Base Construction}

We construct a retrieval index $\mathcal{K}$ from $\mathcal{D}_\text{train}$ by extracting overlapping windows of length $L$ at stride $s$. For each window $\mathbf{x}_i$, we store:
\begin{itemize}
    \item The window itself: $\mathbf{x}_i \in \mathbb{R}^L$
    \item The known future: $\mathbf{f}_i \in \mathbb{R}^H$ (the ground truth following the window)
    \item An embedding: $\mathbf{e}_i = \text{embed}(\mathbf{x}_i)$
\end{itemize}

Embeddings can be extracted from the model's encoder (e.g., Chronos-Bolt's \texttt{[REG]} token) when available, or computed as \emph{functional embeddings}---concatenated z-normalized forecasts from multiple context offsets---for fully black-box models. We use FAISS~\citep{johnson2019faiss} IndexFlatL2 for exact nearest-neighbor search.

\subsection{Distribution-Normalized Re-Forecasting}

Given a test context $\mathbf{x}^*$ with base forecast $\hat{y} = \mathcal{M}(\mathbf{x}^*)$, we retrieve the $k$ nearest neighbors $\{(\mathbf{x}_j, \mathbf{f}_j)\}_{j=1}^k$ from $\mathcal{K}$.

For each retrieved window $\mathbf{x}_j$, we align it to the current context's distribution:
\begin{equation}
    \mathbf{x}_j^{\text{norm}} = \frac{\mathbf{x}_j - \mu_j}{\sigma_j} \cdot \sigma^* + \mu^*
\end{equation}
where $(\mu_j, \sigma_j)$ and $(\mu^*, \sigma^*)$ are the mean and standard deviation of the retrieved and current windows respectively. We then re-forecast: $\hat{y}_j = \mathcal{M}(\mathbf{x}_j^{\text{norm}})$.

\subsection{Consensus-Gated Correction}

The core mechanism computes per-timestep corrections and gates them by directional consensus.

\paragraph{Per-timestep corrections.} For each neighbor $j$ and timestep $t$:
\begin{equation}
    c_j[t] = \hat{y}_j[t] - \hat{y}[t]
\end{equation}

\paragraph{Directional agreement.} The consensus at timestep $t$ is:
\begin{equation}
    a[t] = \left|\frac{1}{k} \sum_{j=1}^k \text{sign}(c_j[t])\right| \in [0, 1]
\end{equation}

When $a[t] = 1$, all neighbors agree on the correction direction. When $a[t] \approx 0$, they are split equally.

\paragraph{Soft consensus gate.} We apply a soft gate:
\begin{equation}
    g[t] = \text{clip}\left(\frac{a[t] - \tau_\text{lo}}{\tau_\text{hi} - \tau_\text{lo}},\; 0,\; 1\right)
\end{equation}
with $\tau_\text{lo} = 0.4$ and $\tau_\text{hi} = 0.8$, producing a gradual transition from no correction (below 40\% agreement) to full correction (above 80\% agreement).

\paragraph{Uncertainty scaling.} To protect easy (low-uncertainty) windows from unnecessary corrections:
\begin{equation}
    s = 0.3 + \frac{0.7}{1 + \exp(-z_u)}, \quad z_u = \frac{u - \bar{u}}{\sigma_u}
\end{equation}
where $u$ is the base model's prediction interval width for the current window, and $\bar{u}, \sigma_u$ are the mean and standard deviation across test windows.

\paragraph{Corrected forecast.}
\begin{equation}
    \tilde{y}[t] = \hat{y}[t] + \beta \cdot s \cdot g[t] \cdot \text{median}(c_1[t], \ldots, c_k[t])
\end{equation}

The hyperparameters ($k$, $\beta$, $\tau_\text{lo}$, $\tau_\text{hi}$) are selected on a single validation split and applied unchanged to all test sets (Section~\ref{sec:validation}).

\subsection{Why Consensus Gating Works}

The consensus gate addresses a fundamental asymmetry in retrieval-based correction: when neighbors provide consistent correction signals (high $a[t]$), the median correction is a reliable estimate of the base model's systematic error at timestep $t$. When neighbors disagree (low $a[t]$), the corrections are noise and should be suppressed.

This differs from standard ensembling, which blends all members unconditionally, and from nearest-neighbor forecasting, which substitutes the base forecast entirely. \method operates as a \emph{sparse, selective corrector}: it modifies only those timesteps where supermajority consensus exists, preserving the base model's predictions everywhere else.

\section{Experimental Setup}
\label{sec:experiments}
%===========================================================================

\subsection{Datasets}

We evaluate on 7 standard long-term forecasting benchmarks: ETTh1, ETTh2, ETTm1, ETTm2~\citep{zhou2021informer}, Weather, Electricity (ECL), and Traffic~\citep{wu2021autoformer}. We use the standard 12/4/rest month split for ETT datasets and 70/10/20 ratio for others, following the Informer~\citep{zhou2021informer} protocol. Prediction horizons: $H \in \{96, 192, 336, 720\}$.

\subsection{Base Models}

\begin{itemize}
    \item \textbf{Chronos-Bolt-Small} (110M params, 512-dim embeddings): encoder-decoder T5-based model~\citep{ansari2024chronos}. Primary evaluation model.
    \item \textbf{Chronos-Bolt-Base} (200M params, 768-dim embeddings): larger variant, validates scaling behavior.
    \item \textbf{TimesFM 1.0} (200M params): decoder-only model from Google~\citep{das2024timesfm}. Uses functional embeddings (Section~\ref{sec:method}). Validates cross-architecture generalization.
\end{itemize}

\subsection{Hyperparameter Selection}
\label{sec:validation}

All hyperparameters are selected on the ETTm2 validation split \emph{only}, then applied unchanged to all 7 test datasets:

\begin{itemize}
    \item Grid search: $k \in \{3, 5, 7\}$, $\beta \in \{0.2, 0.3, 0.4\}$
    \item Selected: $k=5$, $\beta=0.4$ (best validation improvement: +3.85\%)
    \item Gate thresholds $\tau_\text{lo}=0.4$, $\tau_\text{hi}=0.8$ are fixed throughout
    \item KB stride $s=64$, context length $L=512$
\end{itemize}

No hyperparameters are changed between datasets or horizons. The run\_all.py script uses $k=7$, $\beta=0.25$ (original RCAF parameters); we report both configurations.

\subsection{Evaluation Protocol}

\paragraph{Metrics.} MSE on non-overlapping test windows (stride = prediction length $H$).

\paragraph{Statistical tests.} Paired $t$-tests on per-window MSEs. We report raw $p$-values with significance markers ($^{***}p<0.01$, $^{**}p<0.05$, $^*p<0.1$) and Benjamini-Hochberg FDR-corrected significance.

\paragraph{Baselines.} We compare against the vanilla base model (no retrieval) and report the dense ensemble baseline (k=1 nearest-neighbor scalar ensemble, bw=2).

\subsection{Hyperparameter Robustness}

Hyperparameters selected on ETTm2 validation generalize well:
\begin{itemize}
    \item $\beta \in [0.2, 0.4]$: all positive on ETTm2 and Weather
    \item $k \in \{3, 5, 7\}$: all positive on all 3 tested datasets; $k=3$ achieves comparable gains at 5$\times$ overhead (vs.\ 9$\times$ for $k=7$)
    \item Gate thresholds $\tau_\text{lo}, \tau_\text{hi}$: robust in $[0.3, 0.5] \times [0.7, 0.9]$
\end{itemize}

\subsection{Computational Cost}

\method incurs $\sim$15$\times$ inference overhead (with $k=7$) due to $k$ re-forecasts per window. With $k=3$, overhead reduces to $\sim$5$\times$ with comparable accuracy. No training cost. The overhead is dominated by model forward passes and is embarrassingly parallelizable.

\subsection{Failure Analysis: When \method Harms}
\label{sec:failure}

\textbf{Traffic} ($-3.0\%$ average, $-10.4\%$ at $H=96$). Traffic has near-zero values (mean=0.03, std=0.02). Distribution normalization---matching mean and standard deviation---amplifies noise relative to signal, causing all re-forecasts to be unreliable. Despite high consensus (0.918), the corrections are systematically wrong. \textbf{Recommendation:} \method is not suitable for series with near-zero magnitude ($\text{std} < 1$).

\textbf{ETTh2} ($+0.3\%$ average, not significant). ETTh2 exhibits strong non-stationarity with regime changes. Retrieved neighbors from training capture different dynamics despite similar embeddings. The consensus gate correctly produces near-zero corrections, preventing degradation.

\section{Related Work}
\label{sec:related}
%===========================================================================

\paragraph{Retrieval-augmented time series forecasting.}
TS-RAG~\citep{ning2025tsrag} uses a learned Adaptive Retrieval Mixer to fuse retrieved patterns with Chronos-Bolt's internal representations, achieving up to 6.8\% MSE improvement but requiring model internals. RAFT~\citep{han2025raft} trains retrieval-augmented projections. RAF~\citep{tire2024raf} proposes training-free input augmentation (concatenating retrieved contexts), the closest positioning to \method. TimeRAF~\citep{timeraf2024} uses an end-to-end learnable retriever.
\method differs from all prior work in two ways: (1)~it corrects the \emph{output} per-timestep via consensus gating, rather than augmenting the \emph{input} or modifying \emph{internal representations}; (2)~it requires strictly predict()-only access, with no learned components.

\paragraph{Post-hoc forecast revision.}
PIR~\citep{pir2025} proposes model-agnostic post-hoc revision using retrieval and learned uncertainty estimators. \method is fully training-free, using the model's own prediction interval width for uncertainty estimation rather than a learned module.

\paragraph{kNN-augmented inference.}
kNN-LM~\citep{khandelwal2020knnlm} demonstrated that nearest-neighbor retrieval improves language models without training, by interpolating the kNN distribution with the model's output distribution. \method extends this paradigm to continuous time series forecasting, where the key challenge---addressed by consensus gating---is determining \emph{which timesteps} benefit from retrieval correction.

\paragraph{Conformal prediction for time series.}
Conformal prediction methods~\citep{cptc2025,angelopoulos2023conformal} provide calibrated prediction intervals. \method uses a conformal-style residual computation but targets point forecast correction rather than interval estimation.

%===========================================================================
\section{Limitations}
\label{sec:limitations}
%===========================================================================

\begin{enumerate}
    \item \textbf{Traffic-type failures.} \method degrades on near-zero-magnitude series where distribution normalization is unreliable. A simple magnitude check (std $> 1$) prevents deployment in such cases.
    \item \textbf{Point forecast only.} \method improves MSE but does not improve (and may slightly degrade) probabilistic calibration (CRPS, coverage). The per-timestep corrections shift the median toward ground truth but can distort quantile spacing.
    \item \textbf{Computational overhead.} $k=7$ incurs 15$\times$ inference cost. Reducing to $k=3$ (5$\times$) maintains most gains.
    \item \textbf{Univariate evaluation.} Current evaluation is univariate (single target column). Multivariate extension is straightforward but untested.
    \item \textbf{Statistical power.} Long-horizon settings ($H=720$) have few test windows ($N<10$), limiting significance testing. ETTh2, ECL, and Traffic improvements are not statistically significant.
\end{enumerate}

%===========================================================================
\section{Conclusion}
%===========================================================================

We introduced Consensus-Gated Retrieval Correction (\method), a training-free method that improves time series foundation model forecasts by applying per-timestep corrections gated by directional consensus among retrieved neighbors. \method requires only predict() access to the base model, uses no learned parameters, and generalizes across model families (Chronos-Bolt, TimesFM). On 5 of 7 standard benchmarks, \method achieves 4--9\% MSE improvement with statistical significance on the majority of dataset-horizon pairs. The consensus gate is the critical mechanism: it transforms unreliable retrieval correction into a safe, selective refinement tool.
